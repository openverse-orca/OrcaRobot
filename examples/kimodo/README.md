# Kimodo G1 Playback Demo

This directory contains a Kimodo-to-OrcaLab G1 motion generation and playback example:

```text
Text prompt -> Kimodo generates a G1 qpos CSV -> OrcaLab / OrcaGym playback
```

This demo does not run the original trained policy model. It only sends the Unitree G1 MuJoCo qpos sequence generated by Kimodo to OrcaLab for playback.

## Directory

This example lives at:

```text
/home/user/OrcaPlayground/examples/kimodo
```

The scripts automatically locate the OrcaPlayground project root from their own directory. Run commands from the project root:

```bash
cd /home/user/OrcaPlayground
```

Default generated outputs:

```text
examples/kimodo/generated/g1_motion.npz
examples/kimodo/generated/g1_motion.csv
```

## Environments

Kimodo and OrcaLab should use separate conda environments:

```text
Kimodo generation: kimodo
OrcaLab playback:  orcalab
```

The full workflow script activates both environments internally. Default settings:

```bash
KIMODO_ENV=kimodo
ORCALAB_ENV=orcalab
PLAYBACK_ENV=orcalab
KIMODO_ROOT=/home/user/downloads/kimodo
ORCAGYM_ADDR=127.0.0.1:50051
```

Kimodo depends on Meta Llama / LLM2Vec models, which usually require Hugging Face / Meta access or a local model cache. In current testing, the G1 generation workflow can easily hit CUDA OOM on GPUs with less than about 15 GB of VRAM.

## Generate And Play

```bash
cd /home/user/OrcaPlayground

PROMPT="A humanoid robot walks forward and pick up something." \
examples/kimodo/full_text_to_g1_playback.sh
```

The script will:

1. Activate the `kimodo` environment
2. Enter `KIMODO_ROOT`
3. Run `kimodo_gen` to generate `.npz` and `.csv` files
4. Start OrcaLab if the OrcaGym server is not already running
5. Wait for `127.0.0.1:50051`
6. Play the generated CSV

If OrcaLab is already open and you do not want the workflow script to start it:

```bash
PROMPT="A humanoid robot walks forward and pick up something." \
START_ORCALAB=0 \
examples/kimodo/full_text_to_g1_playback.sh
```

## Generate CSV Only

```bash
cd /home/user/OrcaPlayground

PROMPT="A humanoid robot walks forward and pick up something." \
examples/kimodo/generate_g1_csv_with_kimodo.sh
```

Set a custom output name:

```bash
PROMPT="A humanoid robot waves." \
OUTPUT_NAME=wave \
examples/kimodo/generate_g1_csv_with_kimodo.sh
```

Output files:

```text
examples/kimodo/generated/wave.npz
examples/kimodo/generated/wave.csv
```

## Play Existing CSV Only

Make sure OrcaLab / OrcaGym is already running, then run:

```bash
cd /home/user/OrcaPlayground

CSV_PATH=/home/user/OrcaPlayground/examples/kimodo/generated/g1_motion.csv \
examples/kimodo/run_g1_csv_playback.sh
```

To play the default output name:

```bash
OUTPUT_NAME=g1_motion \
examples/kimodo/run_g1_csv_playback.sh
```

If a G1 actor is already in the scene and you do not want the script to spawn another one:

```bash
SPAWN=0 \
CSV_PATH=/home/user/OrcaPlayground/examples/kimodo/generated/g1_motion.csv \
examples/kimodo/run_g1_csv_playback.sh
```

## Start OrcaLab Only

```bash
cd /home/user/OrcaPlayground

examples/kimodo/start_orcalab_server.sh
```

## Kimodo CLI Parameters

`generate_g1_csv_with_kimodo.sh` is roughly equivalent to:

```bash
kimodo_gen "$PROMPT" \
  --model "$MODEL" \
  --duration "$DURATION" \
  --diffusion_steps "$DIFFUSION_STEPS" \
  --num_samples "$NUM_SAMPLES" \
  --output "$OUTPUT_DIR/$OUTPUT_NAME"
```

Common variables:

```bash
PROMPT="A humanoid robot walks forward and pick up something."
MODEL=g1
DURATION=5.0
DIFFUSION_STEPS=100
NUM_SAMPLES=1
SEED=123
OUTPUT_DIR=/home/user/OrcaPlayground/examples/kimodo/generated
OUTPUT_NAME=g1_motion
LOCAL_CACHE=True

MODE=direct
FPS=30
SPEED=1.0
LOOP=1
SPAWN=1
```

## CSV Format

Each CSV row is a G1 MuJoCo qpos:

```text
root xyz + root quaternion + 29 joints = 36 columns
```

In column order:

```text
x y z qw qx qy qz joint_1 ... joint_29
```

## Files
